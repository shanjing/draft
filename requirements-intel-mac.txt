# Intel Mac (x86_64): PyTorch does not ship 2.4+ wheels for macOS Intel (deprecated).
# So we use torch 2.2.x and pin transformers to <4.51 so the stack works without torch 2.4.
#
# Why does the main requirements.txt require torch 2.4?
#   transformers 4.51+ (and accelerate integration) use PyTorch APIs that exist only in 2.4+
#   (e.g. torch.is_autocast_enabled(device_type), type hints using nn.Module). With torch 2.2
#   you get "nn is not defined" or "is_autocast_enabled() takes no arguments".
#
# This file: torch 2.2 + transformers 4.41..<4.51. Embed model all-MiniLM-L6-v2 works.
# Qwen3 embed/reranker (transformers 4.51+) not supported on this stack; use main requirements on Linux/Windows/Apple Silicon.
#
# Install with: pip install -r requirements-intel-mac.txt
